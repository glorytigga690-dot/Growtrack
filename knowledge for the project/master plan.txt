💻 Phase 4: Implementation (Week 5-8)
🔹 Backend (Node.js + Express)
Setup: express, cors, helmet, dotenv, morgan
Auth: JWT generation/verification, bcrypt hashing, role middleware
APIs: Habit/Goal/Mood CRUD, report generation, admin management, subscription sync
Validation: express-validator, sanitization, error handling middleware
Payment Webhooks: Stripe/Razorpay sandbox event handlers
🔹 Frontend Client (Vanilla JS + PWA)
Structure: Modular app.js, feature-based CSS, semantic HTML5
Features: Dashboard layout, logging forms, Chart.js integration, PDF export, settings
PWA: manifest.json, manual service-worker.js (cache-static, network-API), IndexedDB offline queue, install prompt
Offline Logic: Queue unsaved logs → sync on online event → update UI optimistically
🔹 Frontend Admin (Vanilla JS + RBAC)
Protected routes via JWT role check
Tables: User list, subscription status, system metrics
Actions: Suspend/activate users, view audit logs, export CSV/JSON
Analytics: DAU/MAU, plan distribution, feature usage charts

📊 Phase 5: Analytics & Reporting Engine (Week 8-9)
🔹 Growth Score Algorithm
1234
Habit Consistency: (Completed Days / Target Days) × 100
Mood Stability: 100 - (Std Dev of Mood Scores × 10)
Streak Bonus: min(10, current_streak_days)
🔹 Mood-Habit Correlation
Group habit completion rate by mood score (1-5)
Display as scatter/bar chart: "You complete 68% more habits on high-mood days"
Rule-based insight cards for dashboard
🔹 Report Generation
Weekly/Monthly aggregation via backend cron or on-demand endpoint
Client-side PDF export using jsPDF + html2canvas
Embed charts, growth score, top insights, date range
